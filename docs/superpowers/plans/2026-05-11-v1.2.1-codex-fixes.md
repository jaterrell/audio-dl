## Task 1 — [P1] CSRF token + non-loopback bind restriction

**Findings closed:** [P1] Localhost CSRF / drive-by downloads.

**Files:**
- Modify: `audio_dl_ui.py` (token generation in `main()`, embed in HTML, validate in `POST /jobs`, `POST /jobs/{id}/cancel`, `POST /reveal`, and `GET /jobs/{id}/events`)
- Modify: `test_audio_dl_ui.py` (token tests)

### Approach

1. **Token generation:** In `main()`, generate `app.state.csrf_token = secrets.token_urlsafe(32)` before starting uvicorn.
2. **Embed token in HTML:** Add `<meta name="csrf-token" content="...">` to `_INDEX_HTML`, templated in `index()`. Also add a JS const `const CSRF_TOKEN = "...";` at the top of the script block.
3. **Send token from JS:**
   - For `fetch()` calls (POST /jobs, /jobs/{id}/cancel, /reveal): add header `X-Audio-DL-Token: <token>`.
   - For `EventSource` (GET /jobs/{id}/events): append `?token=<token>` to the URL. EventSource can't set custom headers.
4. **Validate on server:** Add a dependency function `_require_csrf(request: Request)` that checks `X-Audio-DL-Token` header OR `token` query param. Returns the token if valid, raises `HTTPException(403)` otherwise. Apply to all state-changing endpoints + SSE.
5. **Bind guard:** In `main()`, if `args.host not in {"127.0.0.1", "localhost", "::1"}` AND `not args.allow_remote`, print an error and exit 1. Add `--allow-remote` flag (action="store_true").

### Steps

- [ ] **Step 1: Write the failing tests** in `test_audio_dl_ui.py`. New class `TestCsrfProtection` covering: (a) `POST /jobs` without token → 403, (b) `POST /jobs` with valid token header → 200, (c) `POST /jobs/{id}/cancel` without token → 403, (d) `POST /reveal` without token → 403, (e) `GET /jobs/{id}/events?token=<valid>` → 200, (f) `GET /jobs/{id}/events?token=<invalid>` → 403, (g) `GET /jobs/{id}/events` without token → 403. **And:** new class `TestBindGuard` covering: (a) `main()` with `--host 0.0.0.0` and no `--allow-remote` exits with code 1 (use `pytest.raises(SystemExit)` and check the code), (b) `--host 0.0.0.0 --allow-remote` allowed. Use `monkeypatch` to stub `uvicorn.run` and `webbrowser.open`.

- [ ] **Step 2: Run tests** — `pytest test_audio_dl_ui.py::TestCsrfProtection test_audio_dl_ui.py::TestBindGuard -v`. Expected: all fail (no token logic yet).

- [ ] **Step 3: Implement token in `main()`** — `import secrets` at top, in `main()` add `app.state.csrf_token = secrets.token_urlsafe(32)`. Add `--allow-remote` argparse flag. Add the loopback guard with a clear error message.

- [ ] **Step 4: Embed token in HTML** — modify `index()` to substitute `__CSRF_TOKEN__` placeholder in `_INDEX_HTML`. Add the placeholder in the HTML `<head>` as `<meta name="csrf-token" content="__CSRF_TOKEN__">` and in the JS as `const CSRF_TOKEN = "__CSRF_TOKEN__";`.

- [ ] **Step 5: Add CSRF dependency function** — `def _require_csrf(request: Request) -> str:` checks `request.headers.get("X-Audio-DL-Token")` first, falls back to `request.query_params.get("token")`. Compares using `secrets.compare_digest` against `app.state.csrf_token`. Raises `HTTPException(403, "Invalid or missing CSRF token")` on mismatch. Use FastAPI `Depends(_require_csrf)` on each protected endpoint.

- [ ] **Step 6: Update JS** — modify `_INDEX_HTML` JS: every `fetch(...)` call sets `headers: {"Content-Type": "application/json", "X-Audio-DL-Token": CSRF_TOKEN}`. The `new EventSource('/jobs/' + job_id + '/events')` becomes `new EventSource('/jobs/' + job_id + '/events?token=' + encodeURIComponent(CSRF_TOKEN))`.

- [ ] **Step 7: Run tests** — `pytest test_audio_dl_ui.py -v`. Expected: all green (existing + new CSRF + bind guard).

- [ ] **Step 8: Run pylint** — `pylint audio_dl_ui.py test_audio_dl_ui.py`. Target 10/10.

- [ ] **Step 9: Commit** with message:
```
feat(ui): CSRF token + non-loopback bind guard

Closes Codex [P1]: malicious websites can no longer trigger downloads
via drive-by fetch to 127.0.0.1. Token generated per launch, embedded
in HTML, required on all state-changing endpoints (header) and SSE
(query param since EventSource can't set headers).

Also: refuse --host outside 127.0.0.1/localhost/::1 unless

LAN or the public internet.
```

## Task 4 — [P2] Bounded SSE queue + progress coalescing

**Findings closed:** [P2] Unbounded event buffering. (NOT the per-subscriber reconnection fix — that's deferred to v1.3.)

**Files:**
- Modify: `audio_dl_ui.py` (`JobState.queue`, `_emit`, `_make_progress_hook`)
- Modify: `test_audio_dl_ui.py` (cap test)

### Approach

- Change `JobState.queue` to `queue.Queue(maxsize=128)`.
- In `_emit`: try `put_nowait`; on `queue.Full`, distinguish event types:
  - Terminal events (`url_started`, `url_completed`, `url_failed`, `job_completed`): block-put with a short timeout, log if it takes >100ms.
  - `progress` events: drop. We already throttle to 5/sec/URL at the hook level so this is rare and harmless.

### Steps

- [ ] **Step 1: Write the failing tests** — `TestQueueBound`: (a) fill the queue past maxsize with progress events, assert it never exceeds 128, (b) on a full queue, a terminal event still gets through (block-with-timeout path).

- [ ] **Step 2: Run tests** — fail (no cap currently).

- [ ] **Step 3: Implement** — change `JobState.queue = field(default_factory=lambda: queue.Queue(maxsize=128))`. Update `_emit(job, event)`: terminal types use `put(event, timeout=1.0)` in a try/except; progress uses `put_nowait` and silently drops on `Full`.

- [ ] **Step 4: Run tests** — green.

- [ ] **Step 5: Commit** with message:
```
fix(ui): bound JOBS queue to 128 events, drop excess progress

Closes Codex [P2] (buffering half): if a client never connects or is
slow, progress events accumulate without backpressure. Cap the queue
at 128. Drop overflow progress events (we already throttle to 5/sec/URL
upstream). Terminal events (url_started/completed/failed, job_completed)
still go through via block-with-timeout.

Note: the related "SSE reconnection loses events" finding requires a
per-subscriber broadcast rewrite and is deferred to v1.3.
```

## Task 5 — [P3] Trio: HTML escape, btoa Unicode, 0.0.0.0 browser open

**Findings closed:** [P3] Self-XSS via unescaped default_output_dir, [P3] `btoa` Unicode throw, [P3] Browser open for `--host 0.0.0.0`.

**Files:**
- Modify: `audio_dl_ui.py`
- Modify: `test_audio_dl_ui.py`

### Approach

1. **HTML escape:** In `index()`, wrap `default_dir` with `html.escape(default_dir, quote=True)` before substitution. `import html` at top.
2. **`btoa` Unicode:** In the JS, replace `btoa(url).replace(/=/g, '')` with a stable hash. Use a simple UTF-8-safe approach: `btoa(unescape(encodeURIComponent(url))).replace(/=/g, '')`. (Avoids dragging in `crypto.subtle` async API for what's basically a DOM id.)
3. **0.0.0.0 open:** In `main()`, when building the browser URL, replace non-routable bind hosts (`0.0.0.0`, `::`) with `127.0.0.1`:
```python
browser_host = "127.0.0.1" if args.host in ("0.0.0.0", "::") else args.host
threading.Timer(0.8, lambda: webbrowser.open(f"http://{browser_host}:{args.port}")).start()
```

### Steps

- [ ] **Step 1: Write the failing tests**
  - `test_default_output_dir_escaped`: pass `--output-dir '<script>alert(1)</script>'` (via `app.state.default_output_dir = ...`), hit `GET /`, assert raw `<script>` is NOT in response body but `&lt;script&gt;` IS.
  - `test_browser_url_rewrites_bind_all`: monkeypatch `webbrowser.open` to capture URL, call `main()` with `--host 0.0.0.0 --allow-remote --no-browser=False`, assert it was called with `127.0.0.1` not `0.0.0.0`.

- [ ] **Step 2: Run tests** — fail.

- [ ] **Step 3: Apply the 3 fixes** as described above.

- [ ] **Step 4: Run tests** — green.

- [ ] **Step 5: Commit** with message:
```
fix(ui): HTML escape default output dir, UTF-8 safe btoa, rewrite 0.0.0.0 browser open

Closes Codex [P3]×3:
- default_output_dir was injected unescaped into an HTML attribute;
  --output-dir '"><script>...' broke markup.
- btoa(url) threw on non-ASCII URLs (IDN, weird copy-paste).
  unescape(encodeURIComponent(...)) makes it UTF-8 safe.
- --host 0.0.0.0 opened browser to http://0.0.0.0:PORT which often
  won't load. Now rewrites to http://127.0.0.1:PORT for the open call.
```

```markdown
## v1.2.1 — 2026-05-11

### Fixed
- **[Security] CSRF token on web UI endpoints.** Random per-launch token,
  required on all state-changing endpoints and the SSE stream. Closes a
  drive-by-download attack class where malicious websites could trigger
  downloads via the localhost-bound server.
- **[Security] Refuse non-loopback bind without `--allow-remote`.** Prevents
  accidentally exposing the UI to LAN or public internet. `--host 0.0.0.0`
  now requires explicit opt-in.
- **HTML escape `--output-dir` in the form.** Self-XSS sink if the launcher
  was passed a crafted directory name.
- **`sanitize_url` exceptions no longer hang the UI row.** Now caught in
  `_run_one`, emit `url_failed`, and let `job_completed` summarize correctly.
- **`/reveal` no longer 500s under concurrent job submission.** `JOBS`
  iteration now snapshots first.
- **Bound SSE queue at 128 events**, drop excess progress events (already
  throttled to 5/sec/URL upstream). Prevents unbounded memory growth from
  slow/disconnected clients.
- **UTF-8 safe URL row hashing.** Pasting URLs with non-ASCII characters
  (IDN domains, accented chars) no longer throws in the browser.
- **Rewrite `0.0.0.0` to `127.0.0.1` in the auto-opened browser URL.** The
  bind-all address often doesn't load in browsers.

### Known issues (deferred to v1.3)
- SSE single-consumer queue: if the browser reconnects mid-job (network
  hiccup, refresh), events may be split between connections. Fix requires
  per-subscriber broadcast architecture.
```
